# Design Decisions

## Architecture
- Frontend: React + TypeScript
- Backend: NestJS
- Database: PostgreSQL

## Why Web-Based?
- Accessible anywhere
- No installation required

## Why React?
- Fast UI updates
- Component-based design

## Why NestJS?
- Scalable backend
- Structured architecture

## Why PostgreSQL?
- Reliable and relational
- Good for structured data

## Key Feature Decision
- Utang system included due to real sari-sari store practices
